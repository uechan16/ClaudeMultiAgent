## 📚 Skills提案機能

### Skillsとは
繰り返し行う作業パターンを再利用可能な「Skill」として登録することで、チーム全体の効率化を図る仕組みです。
登録されたSkillは `.claude/skills/<skill-name>/SKILL.md` に保存され、`/skill-name` で呼び出せます。

### 提案タイミング
以下のような状況でSkill提案を検討してください：
- 同じ作業を2回以上行った
- 他のdevも使えそうな便利な手順を発見した
- 定型的な処理パターンを見つけた

### 提案フォーマット
```bash
send-message.sh manager "【Skill提案】
提案者：dev2
Skill名：[ケバブケースの名前（例：create-api-endpoint）]
説明：[Claudeがいつこのスキルを使うか判断するための説明]
手順：
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]
備考：[その他の補足情報]"
```

### 提案例
```bash
send-message.sh manager "【Skill提案】
提案者：dev2
Skill名：create-api-endpoint
説明：標準的なCRUD APIエンドポイントを作成する。新しいリソースのAPIが必要な時に使用。
手順：
1. ルーティング定義を追加
2. コントローラーを作成
3. バリデーションミドルウェアを設定
4. サービス層のロジックを実装
5. エラーハンドリングを追加
6. APIドキュメントを更新
備考：認証が必要な場合はミドルウェアを追加"
```
