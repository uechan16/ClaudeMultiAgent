## 📚 Skills提案機能

### Skillsとは
繰り返し行う作業パターンを再利用可能な「Skill」として登録することで、チーム全体の効率化を図る仕組みです。
登録されたSkillは `.claude/skills/<skill-name>/SKILL.md` に保存され、`/skill-name` で呼び出せます。

### 提案タイミング
以下のような状況でSkill提案を検討してください：
- 同じ作業を2回以上行った
- 他のdevも使えそうな便利な手順を発見した
- 定型的な処理パターンを見つけた

### 提案フォーマット
```bash
send-message.sh manager "【Skill提案】
提案者：dev3
Skill名：[ケバブケースの名前（例：tech-research）]
説明：[Claudeがいつこのスキルを使うか判断するための説明]
手順：
1. [ステップ1]
2. [ステップ2]
3. [ステップ3]
備考：[その他の補足情報]"
```

### 提案例
```bash
send-message.sh manager "【Skill提案】
提案者：dev3
Skill名：tech-research
説明：新技術・ライブラリの選定調査を行う。技術選定が必要な時に使用。
手順：
1. 候補技術を3-5個リストアップ
2. 各技術の公式ドキュメントを確認
3. GitHubスター数・更新頻度を調査
4. 類似プロジェクトでの採用事例を調査
5. 比較表を作成
6. 推奨技術と理由をまとめる
備考：調査結果はdocs/research/に保存"
```
